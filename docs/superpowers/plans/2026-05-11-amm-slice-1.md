# AMM Slice 1 ‚Äî Multi-asset schema + terms + admin USDC credit

> **For agentic workers:** Steps use checkbox (`- [ ]`) syntax. Slice 1 of the [RPOW Pool design spec](../specs/2026-05-11-amm-design.md).

**Goal:** Add USDC as a second database-tracked asset, add the AMM terms-acceptance flag, and expose both via `/me`. Add an admin endpoint to credit USDC for testing before the Solana deposit indexer ships.

**Architecture:** Two new columns on `users` (`usdc_base_units`, `amm_terms_accepted_at`), two new env vars (`AMM_ALLOWED_EMAILS` for alpha gating + `AMM_ADMIN_EMAILS` for admin endpoints), one new route module (`apps/server/src/routes/amm/index.ts`) registering two endpoints (`POST /amm/accept-terms`, `POST /amm/admin/credit-usdc`), and a small extension to `/me`.

**Alpha gate:** `AMM_ALLOWED_EMAILS` defaults to `frk314@gmail.com` ‚Äî only that account can hit any `/amm/*` endpoint at launch. Every AMM endpoint runs the allowlist check FIRST (returns 403 NOT_ALLOWED otherwise), THEN the terms-accepted check. `/me` always works (no allowlist gate); for non-allowed users, the `usdc_base_units` field reads `'0'` (zeroed because they've never been credited).

**Tech Stack:** Postgres 17, Fastify 4, zod, vitest.

## Task 1: Migration 019

```sql
-- AMM slice 1: add USDC balance + terms-acceptance flag to users.
ALTER TABLE users
  ADD COLUMN usdc_base_units BIGINT NOT NULL DEFAULT 0
    CHECK (usdc_base_units >= 0),
  ADD COLUMN amm_terms_accepted_at TIMESTAMPTZ NULL;
```

Test asserts both columns exist with the expected types + the CHECK constraint rejects negative USDC.
